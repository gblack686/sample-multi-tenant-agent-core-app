# Strands Agents SDK POC — UC-02 Micro-Purchase

## Context

EAGLE currently uses Claude Agent SDK (`claude_agent_sdk`) which spawns a Node.js
subprocess per query. We're evaluating Strands Agents SDK as a potential replacement
(in-process, native Bedrock). This spike proves the SDK works with the existing
oa-intake SKILL.md prompt and passes the same eval assertions.

## Scope

- Branch: `spike/strands-poc`
- 1 new file, 1 modified file
- Mirrors Test 21 (`test_21_uc02_micro_purchase`) from the existing eval suite
- Same skill prompt, same test prompt, same 5 indicators, same pass threshold (>= 3/5)

## Changes

### 1. `server/requirements.txt` — add dependency

Add after the `claude-agent-sdk` line:

```
strands-agents>=1.0.0
```

### 2. `server/tests/test_strands_poc.py` — new file

Single sync test function that:

1. **Loads skill** from `eagle_skill_constants.SKILL_CONSTANTS["oa-intake"]` (reuses existing loader — no changes)
2. **Builds system prompt** — identical tenant context + skill body as Test 21
3. **Constructs Strands Agent**:
   ```python
   model = BedrockModel(
       model_id="us.anthropic.claude-haiku-4-5-20251001-v1:0",
       region_name="us-east-1",
   )
   agent = Agent(model=model, system_prompt=system_prompt, callback_handler=None)
   ```
4. **Invokes synchronously** — `result = agent(prompt)` (no subprocess, no async generator)
5. **Extracts text** — `str(result).lower()`
6. **Asserts 5 indicators** — identical keyword checks from Test 21:
   - `micro_purchase`, `threshold`, `purchase_card`, `streamlined`, `far_reference`
   - Pass: >= 3 found

Key design decisions:
- **Sync invocation** (`agent()`) not async — simplest possible; no pytest-asyncio needed
- **No TraceCollector** — Strands returns `AgentResult` directly; `str(result)` extracts text
- **Shared nothing** with existing SDK code — this is a standalone spike file
- **`callback_handler=None`** suppresses Strands' default console printing
- Runnable both as `pytest tests/test_strands_poc.py -v -s` and `python tests/test_strands_poc.py`

## Verification

```bash
# 1. Create branch
git checkout -b spike/strands-poc

# 2. Install
cd server && pip install "strands-agents>=1.0.0"

# 3. Verify imports
python -c "from strands import Agent; from strands.models import BedrockModel; print('OK')"

# 4. Run the POC test (requires AWS creds with Bedrock access)
python -m pytest tests/test_strands_poc.py -v -s

# 5. Expected: PASS with >= 3/5 indicators
```

## What this proves

- Strands SDK works with Bedrock in our AWS account
- Existing SKILL.md prompts are portable (zero changes to eagle-plugin/)
- `eagle_skill_constants.py` loader is SDK-agnostic
- No subprocess overhead — direct in-process boto3 `converse` call

## What this does NOT prove (future spikes)

- Multi-agent (supervisor → subagent via agents-as-tools)
- Streaming to frontend (SSE via `stream_async()`)
- Session persistence
- Hot-reload of prompts (per-request agent construction)
- Tool integration (`@tool` with Read/Glob/Bash)
